SoV0: a fixed-supply zero-fee cryptocurrency

SoV0 is a cryptocurrency optimized purely for being a store-of-value. SoV0â€™s design is simple: just a publicly-maintained text file documenting who owns how many units of some abstract asset, plus a rudimentary transaction system. SoV0 is too slow to be a general purpose payments platform, has no smart contract support, and certainly can't function as a currency. SoV0 intentionally adopts this limited functionality in order to achieve 0 supply growth and 0 fees (unlike all existing assets), with the long-term goal of becoming a singularly compelling coordinated value store.

1. Generate a SoV0 address






Your SoV0 address: 

  • Write down your username & password, since address generation is done entirely on your computer (using libsodium.js) & username/password are not sent/stored anywhere.
  • Anyone who knows (or guesses) your username & password will have access to the assets stored at your address. When SoV0 goes live we'd recommend using a strong randomly generated password (currently, SoV0 is in testnet and all assets will be removed in a few weeks).
  • The same username + password will always generate the same address.

3. Going beyond

Once you've generated an address and requested some balance, you should receive some SoV0 testnet assets at your address the following Monday.

If you want to check how much balance your address has: (1) go to the pinned tweet at https://twitter.com/the_SoV0, (2) download the state file via the link a few replies down, and (3) ctrl+f for your address in that file.



If your address doesn't appear in the file, then one of these things happened: (1) your balance request was submitted too late to have been included in this period and you'll get some assets next period, (2) your balance request had some issues (e.g. improper format, incorrect state) and couldn't be included, or (3) the block producer otherwise failed to include the request.

Once you've confirmed you have some assets, you can try sending a transaction. To do so, see this section of the whitepaper. Note that these assets are purely for testing purposes, and will be removed once this public testnet period concludes.

You may also want to monitor the weekly state update process by checking that the block producer is behaving properly, by following the instructions in this and this section of the whitepaper.
